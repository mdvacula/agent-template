## Setup steps (run once, in order)

```bash
npx @fission-ai/openspec@latest init --tools opencode,pi  # creates .opencode/ + .pi/ skills/commands
git config --add remote.origin.fetch '+refs/notes/*:refs/notes/*'  # auto-fetch notes
# write opencode.json with MCP hub config
# copy agents from agent-kit into .opencode/agents/ and .pi/agents/
```

## Git Notes quick reference

```bash
# Attach note (REQUIRED after every task commit)
git notes --ref=agent-log add -m '{
  "agent":    "opencode",
  "model":    "anthropic/claude-sonnet-4-5",
  "task_id":  "TASK-42",
  "spec_ref": "auth/jwt-flow",
  "summary":  "Implement JWT refresh flow"
}'

# Read note on HEAD
git notes --ref=agent-log show HEAD

# Read note on any commit
git notes --ref=agent-log show <hash>

# Browse all annotated commits
git log --show-notes=agent-log --oneline

# Push notes
git push origin refs/notes/agent-log

# Configure auto-fetch (one-time per clone)
git config --add remote.origin.fetch '+refs/notes/*:refs/notes/*'
```

## MCP tool quick reference

| Tool | Purpose |
|------|---------|
| `fetch_tasks(status="pending")` | Get work queue |
| `fetch_tasks(id="<id>")` | Get single task |
| `update_task_status(id, "in-progress")` | Claim a task |
| `update_task_status(id, "completed")` | Close a task |
| `sync_task(id, title, status, metadata)` | Upsert task + metadata |

## Agent session loop

1. `fetch_tasks(status="pending")` — health check + queue
2. Parse Living Spec → `sync_task` for new requirements
3. Pick top task → `update_task_status(in-progress)`
4. Implement
5. `git commit -m "..."`
6. `git notes --ref=agent-log add -m '{"agent":"...","task_id":"...","summary":"..."}'`
7. Tests pass, build green
8. `update_task_status(completed)`
9. Tick requirement in `openspec/specs/living-spec.md`
10. Zero-bloat check: `git diff --cached --name-only` (no task artifacts)
11. `git push origin HEAD refs/notes/agent-log`

## Zero-bloat rule

**Only code + Living Spec in Git's file tree. Task state in hub. Observability in git notes.**

These NEVER appear in a commit:
- `*.tasks.jsonl`, `tasks.db`, `tasks.sqlite`
- Task markdown sidecars outside `openspec/specs/`
- `.beads/` or any Beads artifacts

## Do not track

```gitignore
openspec/changes/
openspec/changes/*/proposal.md
openspec/changes/*/design.md
openspec/changes/*/tasks.md
*.tasks.jsonl
tasks.db
tasks.sqlite
.tasks/
.beads/
```

- [ ] `opencode.json` contains `mcp.task-hub` config
- [ ] `fetch_tasks(status="pending")` returns valid response
- [ ] `AGENTS.md` exists at repo root
- [ ] `openspec/specs/living-spec.md` exists
- [ ] After first task commit: `git notes --ref=agent-log show HEAD` returns JSON
- [ ] `.gitignore` includes zero-bloat rules
- [ ] `git status` after a full task cycle: no task artifacts in diff
